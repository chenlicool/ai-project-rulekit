# AGENTS_TEST.md (测试法典)

版本号: v1.0
生效时间: YYYY-MM-DD
适用对象: 所有代码提交、功能验证、Bug 修复

## 1. 适用范围 (Scope)
所有项目功能的验证流程、Mock 数据定义、测试用例编写。

## 2. 强制规则 (MUST)
- **AI 审计**：AI 生成或修改的代码默认视为“未可信”。
- **Mock 优先**：前端开发前必须先定义 Mock 数据，避免依赖不稳定后端。
- **关键路径覆盖**：核心链路必须有回归测试（手动或自动）。
- **组件先行**：复杂组件需在隔离环境中先验证。

## 3. 禁止行为 (MUST NOT)
- 禁止未通过测试直接合并代码。
- 禁止使用生产数据进行开发测试。
- 禁止忽略 Lint / 编译错误。

## 4. 允许例外 (EXCEPTION)
- 纯视觉样式微调可放宽测试流程，但需人工 Review 确认。

## 5. 违规处理 (ENFORCEMENT)
- 未通过测试的 AI 改动禁止合并。
- 每次重大改动后必须输出测试记录：
  - 时间：YYYY-MM-DD HH:MM
  - 触发命令：
  - 覆盖路径：
  - 结果：
  - 异常：无/简述
